## Findings

### P0 launch blocker

None found in this pass.

### P1 should fix before launch

1. **Hero overlay contrast on staging is weak.**
   - Status: fixed in this branch, not deployed yet.
   - Details: the homepage hero overlay used `bg-brand-mahogany/92`, which was not generated by the current Tailwind setup and rendered transparent on staging. This made ivory text sit directly on the photo.
   - Fix: changed the overlay to solid `bg-brand-mahogany`.
   - Recommended branch: `codex/staging-qa-pass`.

2. **Fresh Lion's Mane cart uses the staged fallback on live staging.**
   - Status: observed in this QA branch; follow-up work is documented in `docs/qa/staging-medusa-cart-path.md`.
   - Details: after adding Fresh Lion's Mane, the cart showed the safe staged browser cart fallback because the product data handed to the browser bridge did not include a Medusa variant ID. This is safe, but it means live staging is not exercising the full Medusa-backed cart path for that product.
   - Recommended branch: `codex/staging-cart-medusa-variant-sync`.

### P2 polish

1. **Browser requested `/favicon.ico` and staging returned 404.**
   - Status: fixed in this branch, not deployed yet.
   - Fix: added Next metadata icon entries pointing at the existing brand PNG.

2. **Mobile hero and checkout should receive one more manual review after redeploy.**
   - Status: follow-up recommended.
   - Details: CDP measurement reported no horizontal overflow at 390 px, but screenshots show the hero and checkout are visually tight on mobile.
   - Recommended branch: `codex/mobile-polish-pass`.

3. **Some staging copy still references future Medusa checkout behavior.**
   - Status: follow-up recommended.
   - Details: this is acceptable for staging, but public launch copy should be reviewed once the final checkout path is chosen.

4. **Product CTA mix needs owner confirmation before launch.**
   - Status: owner review recommended.
   - Details: Fresh Lion's Mane currently presents add-to-cart and notify/availability CTAs. This may be correct for seasonal/launch stock testing, but launch behavior should match actual availability.

### P3 future improvement

1. Add an automated staging smoke test that captures route status, console errors, missing assets, and checkout safety signals.
2. Add Lighthouse or equivalent accessibility/performance review after domain-based staging and HTTPS are available.
3. Replace interim raster/favicon usage with final designer-provided SVG/favicon package before production launch.
